1차 프론트 API
좋습니다. 그러면 Admin 권한 제어는 뒤로 미루고, 지금은 프론트가 바로 연결할 수 있도록
GreenLink 1차 MVP 프론트 연결용 API 응답 최종 정리를 하겠습니다.
기준은 현재 구현 상태입니다.
인증 방식: JWT
요청 Header: Authorization: Bearer {accessToken}
공통 응답 형식: ApiResponse<T>
DTO 구조: 통합 DTO 방식
Admin API: 프론트 일반 사용자 화면 연결 대상에서 제외

1. 공통 응답 형식

모든 API 응답은 아래 구조로 통일합니다.

성공 응답
{
"success": true,
"message": "요청 성공 메시지",
"data": {}
}

실패 응답
{
"success": false,
"message": "오류 메시지",
"data": null
}
프론트에서는 기본적으로 이렇게 처리하면 됩니다.

1차 프론트 API

1

success = true

→ data 사용

success = false → message를 사용자에게 표시

2. 인증 방식

로그인 이후 프론트는 accessToken 을 저장하고, 인증이 필요한 API마다 Header에 넣습니
다.

Authorization: Bearer {accessToken}

예시:
GET /api/users/me
Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...

인증이 필요한 API에서 토큰이 없거나 잘못되면 대체로 아래처럼 처리됩니다.
{
"success": false,
"message": "인증이 필요합니다.",
"data": null
}
